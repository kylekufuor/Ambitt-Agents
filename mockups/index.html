Ambitt Agents · Portal Redesign · Mockups

A quiet system, only what the client needs.

The portal isn't where the work happens — that's email, WhatsApp, the inbox. The portal is where a client comes to see what their agent is doing for them, plug in the keys, and say yes or no to the things the agent wants to do next. It should feel like a single calm dashboard a consultant might have prepared.

MVP scope locked at 4 surfaces. The four boxed below are what we ship for v1 clients. The four greyed at the bottom are exploration — we built mockups to think them through, but we're not building them as routes yet. When a real client asks for one, we'll know it earned the space.

— Explored · not building unless a real client asks for it

These mockups exist so we've thought through the shape. Shipping them now would bloat the surface area for no client benefit — and every page we ship is a page we have to maintain. The principle: only what the client needs to use weekly.

Deferred No client has asked

Activity log

The forensic view. Overview's feed covers the daily question (what did Arthur do today?). A searchable, paginated full log is operator-territory — clients don't audit their consultant's work hour-by-hour. Build when one actually requests an export.

Deferred Operator-tuned for now

Settings (incl. KPI picker)

Schedule, voice, autonomy, KPIs — all of this lives on the agent's overview page already (or in the dashboard for operator-side tuning). A dedicated Settings tab adds clicks without adding control. The KPI picker is a great idea — we'll wire it onto Overview when KPIs themselves ship.
